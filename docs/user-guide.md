# 合伙记 - 使用说明

## 简介

合伙记是一款面向合作项目团队的 Web 端财务协作系统 Demo。它帮助团队管理项目活动、记录收支、自动计算利润、对账供应商付款、管理发票，并支持 Excel 数据导入导出。

## 快速开始

### 1. 安装依赖

```bash
cd openspec-demo
npm install
```

### 2. 启动服务

```bash
npm start
```

服务默认运行在 `http://localhost:3000`，打开浏览器即可访问。

### 3. 运行测试

项目包含 **143 个测试用例**，分为单元测试和系统测试两大类。

```bash
# 运行全部测试
npm test

# 仅运行单元测试（7个文件，约80个用例）
npm run test:unit

# 仅运行系统测试（4个文件，约63个用例）
npm run test:system
```

#### 单元测试覆盖

- **用户服务** - 8种角色创建、权限矩阵验证、用户状态管理
- **项目服务** - 项目创建、成员添加、分成比例校验
- **交易服务** - 收支记录、金额/类型校验、筛选与统计
- **利润服务** - 净利润计算、亏损处理、按比分配、情景模拟
- **供应商服务** - 供应商管理、付款记录、对账差异检测
- **发票服务** - 上传发票、模拟OCR、项目隔离
- **Excel服务** - 批量导入（含失败行报告）、报表导出

#### 系统测试覆盖

- **完整工作流** - 从创建项目到导出报表的全流程验证
- **错误处理** - 各模块404/400错误场景、参数校验
- **用户管理API** - 用户CRUD、角色切换、权限查询
- **导入导出API** - 批量导入验证、导出报表完整性

## 功能使用指南

### 项目管理

1. **创建项目**：在「项目管理」标签页填写项目名称、描述和创建者ID，点击「创建」。
2. **添加成员**：输入项目ID、用户ID、选择角色（管理员/编辑/只读），设置分成比例。
3. **设置分成比例**：输入项目ID和比例JSON，例如 `{"user-1":0.6,"user-2":0.4}`，比例之和必须等于1。

### 收支记录

1. **记录交易**：选择项目ID、类型（收入/支出）、金额、日期、描述和分类，点击「记录」。
2. **查询统计**：输入项目ID和日期范围，点击「统计」查看总收入、总支出和净额。
3. **查询明细**：点击「查询明细」查看交易列表。

### 利润计算

1. **计算利润**：输入项目ID，点击「计算」，系统会自动按预设比例分配利润。
2. **情景模拟**：输入额外收入或支出，模拟不同情景下的利润变化。

### 供应商对账

1. **添加供应商**：输入项目ID、供应商名称和联系方式。
2. **记录付款**：输入供应商ID、金额、状态（待付/已付/逾期）和到期日。
3. **开始对账**：系统会自动检测到期未付的款项并标出差异。

### 发票管理

1. **上传发票**：输入项目ID和文件名，系统会模拟OCR识别金额和日期。
2. **查询发票**：输入项目ID查看已上传的发票列表。

### 导入导出

1. **导入交易**：输入项目ID和JSON格式的交易数据，系统会逐行验证并导入。
2. **导出报表**：输入项目ID，系统会生成包含交易、供应商、发票的完整报表。

## 数据格式示例

### 导入交易JSON格式

```json
[
  {"type":"income","amount":5000,"date":"2024-01-15","description":"客户A付款","category":"service"},
  {"type":"expense","amount":1200,"date":"2024-01-20","description":"材料费","category":"material"}
]
```

### 分成比例JSON格式

```json
{"user-1":0.6,"user-2":0.4}
```

## 注意事项

- 本 Demo 使用内存存储，重启服务后数据会丢失。
- OCR 功能为模拟实现，实际生产环境可接入 Tesseract 或云OCR服务。
- Excel 导入导出当前为JSON格式模拟，生产环境可集成 SheetJS (xlsx) 库。
